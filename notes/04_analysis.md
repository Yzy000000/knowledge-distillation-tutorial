# 知识蒸馏实验结果深度分析

## 1. 实验设置回顾

我们基于 CIFAR-10 数据集，使用参数量约 **118.7 万**的教师模型（DeepNN）和约 **26.8 万**的学生模型（LightNN），通过以下五种方式训练学生：

| 方法 | 描述 |
|------|------|
| 基线 | 仅交叉熵损失（CE） |
| 输出蒸馏 (KD) | CE + KL 散度（软标签），温度 T=2，损失权重 0.25/0.75 |
| 余弦损失 (Cosine) | CE + 余弦嵌入损失（对齐隐藏表示），隐藏层经池化降维 |
| 特征图 MSE (RegressorMSE) | CE + MSE 损失（对齐特征图），学生增加可训练回归器 |

## 2. 实验结果汇总

| 方法 | 测试准确率 (%) |
|------|----------------|
| 教师 (DeepNN) | 74.84 |
| 学生基线 (CE) | 70.55 |
| 输出蒸馏 (KD) | 70.13 |
| 余弦损失 (Cosine) | 70.83 |
| 特征图 MSE (RegressorMSE) | 71.11 |

## 3. 结果分析

### 3.1 教师与学生基线差距
教师准确率比学生高约 **4.3 个百分点**，验证了更大容量模型能学习到更丰富的特征。学生基线（70.55%）作为后续蒸馏的基准。

### 3.2 输出蒸馏（KD）效果不佳
KD 不仅未能提升，反而略低于基线（70.13%）。可能原因：
- **超参数未优化**：温度 T=2，损失权重 0.25/0.75 可能不适用于本任务。文献中通常对温度、权重进行网格搜索。
- **学生容量不足**：轻量级网络难以拟合教师输出的完整分布，强行拟合可能引入噪声，干扰分类任务。
- **师生输出分布差异**：教师与学生的 logits 分布可能差异较大，软标签提供的“暗知识”对学生帮助有限。
- **训练动态**：蒸馏损失可能过早主导训练，导致分类损失未充分优化。

### 3.3 余弦损失带来微弱提升
通过池化降维后对齐隐藏表示，准确率提升至 70.83%（+0.28%）。表明中间表示对齐有一定帮助，但提升有限，可能因为：
- **池化操作损失信息**：将 2048 维降至 1024 维，丢弃了部分空间/通道信息。
- **余弦损失仅约束方向**：只要求向量方向一致，不要求数值接近，约束较弱。
- **缺少适配层**：直接对齐可能导致学生强行模仿，缺乏灵活性。

### 3.4 特征图 MSE 损失效果最佳
引入可训练回归器后，准确率达到 71.11%（+0.56%），验证了“基于提示的蒸馏”的有效性：
- **可训练适配层**：回归器（卷积层）将学生特征图从 16 通道提升到 32 通道，为学生提供了额外的学习能力，不必直接复制教师表示。
- **逐像素 MSE 损失**：在特征图层面进行精细匹配，保留了空间结构，更有利于知识迁移。
- **平衡训练**：MSE 损失权重 0.25，分类损失权重 0.75，使得学生既学习特征表示又保持分类能力。

## 4. 不同蒸馏策略的适用性讨论

### 4.1 输出蒸馏 (KD)
- **优点**：实现简单，不改变模型结构，可直接用于任何输出维度相同的网络。
- **缺点**：当学生容量远小于教师时，可能难以拟合软标签；需要精细调参（温度、权重）。
- **适用场景**：教师与学生容量差距较小，或学生已接近教师性能时，可用于微调。

### 4.2 余弦损失（中间表示对齐）
- **优点**：无需添加额外参数，仅需降维匹配维度。
- **缺点**：约束较弱，降维可能损失信息。
- **适用场景**：当学生隐藏表示维度与教师相近时，可作为轻量级蒸馏手段。

### 4.3 特征图 MSE（基于提示的蒸馏）
- **优点**：通过适配层增加灵活性，在特征空间精细匹配，效果显著。
- **缺点**：增加了少量参数（回归器），需要修改模型结构。
- **适用场景**：学生容量较小，且希望最大化知识迁移效果时，推荐使用。

## 5. 超参数影响的思考

本实验中，所有蒸馏方法的损失权重均采用 0.25（蒸馏）/0.75（CE），温度 T=2。这些值未进行调优，可能不是最优选择。例如：
- 增大温度可使软标签更平滑，但过高的温度可能模糊类别差异。
- 调整权重可以改变蒸馏与分类任务的平衡。

未来可尝试对温度、权重进行网格搜索，以进一步提升效果。

## 6. 结论

- 在本实验中，**特征图 MSE 损失（带回归器）** 取得了最佳的学生性能（71.11%），证明了通过可训练适配层在特征层面进行知识迁移的有效性。
- 输出蒸馏效果不佳，提示在小容量学生场景下，中间特征蒸馏可能优于输出蒸馏。
- 余弦损失虽简单，但提升有限，主要受限于降维和弱约束。
- 选择合适的蒸馏策略应综合考虑学生容量、模型结构、超参数调优等因素。

## 7. 未来工作建议

- 调整温度、损失权重，重新评估输出蒸馏性能。
- 尝试其他适配层结构（如全连接、注意力机制）进行特征匹配。
- 在更大规模数据集（如 ImageNet）上验证结论。
- 结合多种蒸馏损失（如输出蒸馏 + 特征图 MSE），观察协同效果。
